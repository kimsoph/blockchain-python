# 블록체인 기본 원리

> 초보자를 위한 블록체인 핵심 개념 정리

---

## 1. 블록체인이란?

**한 줄 정의**: 여러 컴퓨터가 공유하는 "수정 불가능한 장부"

### 비유로 이해하기

```
일반 장부 (중앙화)          블록체인 (분산화)
    [은행]                  [A] [B] [C] [D]
      ↓                      ↓   ↓   ↓   ↓
  모든 기록              모두 같은 장부 보유
  은행이 관리            누구도 혼자 수정 불가
```

**핵심**: 모든 참여자가 같은 기록을 갖고 있어서, 누군가 몰래 바꾸면 즉시 들통남

---

## 2. 블록(Block)이란?

**한 줄 정의**: 거래 기록을 담은 "상자"

### 블록의 구성

```
┌─────────────────────────┐
│ Block #3                │
├─────────────────────────┤
│ 이전 블록 해시: abc123  │  ← 이전 블록과 연결
│ 타임스탬프: 2024-01-01  │
│ 거래 내용:              │
│   - Alice → Bob: 10원   │
│   - Bob → Charlie: 5원  │
│ Nonce: 12345            │
│ 해시: def456            │  ← 이 블록의 지문
└─────────────────────────┘
```

**예시**: 택배 상자에 송장 번호(해시)가 붙어있고, 이전 상자 번호도 적혀있는 것

---

## 3. 체인(Chain)이란?

**한 줄 정의**: 블록들이 순서대로 연결된 "사슬"

```
[Block 0] → [Block 1] → [Block 2] → [Block 3]
 Genesis     hash:abc     hash:def     hash:ghi
             prev:000     prev:abc     prev:def
```

**왜 체인인가?**
- 각 블록은 이전 블록의 해시를 저장
- 중간 블록 수정 → 뒤의 모든 해시가 깨짐 → 조작 즉시 발각

**예시**: 도미노처럼 하나 건드리면 뒤가 다 무너짐

---

## 4. 해시(Hash)란?

**한 줄 정의**: 데이터의 "지문" (고유 식별자)

### 해시의 특징

| 특징 | 설명 |
|------|------|
| 고정 길이 | 입력 크기와 무관하게 항상 64자 |
| 단방향 | 해시 → 원본 복원 불가능 |
| 민감성 | 한 글자만 바꿔도 완전히 다른 해시 |

### 예시

```python
"안녕하세요" → a1b2c3d4e5f6...  (64자)
"안녕하세욧" → z9y8x7w6v5u4...  (완전히 다름!)
```

**비유**: 사람의 지문 - 똑같은 지문을 가진 사람은 없음

---

## 5. 채굴(Mining)이란?

**한 줄 정의**: 조건에 맞는 해시를 찾는 "노가다"

### 작업 증명 (Proof of Work)

```
목표: 해시가 "0000"으로 시작하는 숫자(Nonce) 찾기

시도 1: Nonce=1 → 해시: a3f2... ❌
시도 2: Nonce=2 → 해시: 8b1c... ❌
...
시도 50000: Nonce=50000 → 해시: 0000abc... ✓ 성공!
```

**왜 이런 짓을?**
- 블록 생성에 "비용"(시간, 전기)이 들게 함
- 조작하려면 엄청난 컴퓨팅 파워 필요
- 정직하게 채굴하는 게 이득

**보상**: 채굴 성공 시 코인 지급 (비트코인: 현재 6.25 BTC)

### Nonce 심층 이해: PoW 채굴의 본질

**Nonce = Number used ONCE** (한 번만 사용되는 숫자)

#### 왜 Nonce가 핵심인가?

블록의 해시는 다음 데이터로 계산됩니다:

```
해시 = SHA256(이전해시 + 타임스탬프 + 거래내용 + Nonce)
```

여기서 **Nonce만 자유롭게 바꿀 수 있는 값**입니다:
- 이전 해시: 고정 (이전 블록에서 결정됨)
- 타임스탬프: 고정 (현재 시간)
- 거래 내용: 고정 (포함할 트랜잭션들)
- **Nonce: 변경 가능** ← 이것만 조절!

#### 채굴 = Nonce 찾기

```
┌─────────────────────────────────────────────────────┐
│                    채굴 과정                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  목표: 해시가 "0000"으로 시작해야 함 (난이도=4)      │
│                                                     │
│  [고정값]                    [변경값]               │
│  ┌─────────────────────┐    ┌─────────┐            │
│  │ 이전해시: abc123    │    │ Nonce=? │            │
│  │ 시간: 2024-01-01    │ +  │         │ → 해시계산 │
│  │ 거래: Alice→Bob     │    └─────────┘            │
│  └─────────────────────┘                           │
│                                                     │
│  Nonce=1    → 해시: 7f3a2b... ❌ (0으로 안시작)     │
│  Nonce=2    → 해시: e5c8d1... ❌                    │
│  Nonce=3    → 해시: 2a9f4e... ❌                    │
│  ...                                               │
│  Nonce=78803 → 해시: 0000c2... ✓ 성공!             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 왜 "한 번만 사용"인가?

```
Block #5 채굴 시: Nonce=78803으로 성공
Block #6 채굴 시: 같은 78803이 통할까? → ❌ 절대 안됨!

이유:
- Block #6의 "이전 해시"가 다름 (Block #5의 해시)
- Block #6의 "거래 내용"이 다름
- Block #6의 "타임스탬프"가 다름

→ 매번 새로운 Nonce를 처음부터 찾아야 함
→ 그래서 "Number used ONCE"
```

#### 난이도와 Nonce의 관계

| 난이도 | 조건 | 평균 시도 횟수 | 시간 |
|--------|------|---------------|------|
| 1 | 0으로 시작 | ~16회 | 즉시 |
| 2 | 00으로 시작 | ~256회 | 즉시 |
| 4 | 0000으로 시작 | ~65,536회 | 수초 |
| 6 | 000000으로 시작 | ~1,677만회 | 수분 |
| 비트코인 | 0 약 19개 | ~수조회 | ~10분 |

**핵심 공식**: 난이도 N → 평균 16^N 번 시도 필요

#### 비유: 자물쇠 비밀번호 찾기

```
상황: 4자리 자물쇠 (0000~9999)
조건: 비밀번호가 "0000"으로 시작해야 열림

방법: 0000, 0001, 0002... 하나씩 시도
      → 언젠가 맞는 번호 발견
      → 이게 채굴!

Nonce = 시도하는 번호
해시 = 자물쇠의 반응 (열림/안열림)
난이도 = 자물쇠의 복잡도
```

#### 결론: Nonce 찾기 = PoW의 본질

```
┌────────────────────────────────────────┐
│  "채굴한다" = "Nonce를 찾는다"          │
│                                        │
│  • 수학 문제를 푸는 게 아님            │
│  • 그냥 숫자를 1씩 올려가며 대입       │
│  • 조건 맞는 해시 나올 때까지 반복     │
│  • 운 + 컴퓨팅 파워 = 채굴 성공        │
└────────────────────────────────────────┘
```

## 6. 분산 네트워크

**한 줄 정의**: 여러 컴퓨터가 같은 장부를 유지

```
        ┌─────┐
        │노드A│ ←─┐
        └─────┘   │
   ┌─────┐    ┌─────┐
   │노드B│ ←→ │노드C│
   └─────┘    └─────┘
        │         │
        └──→ ┌─────┐
             │노드D│
             └─────┘

모든 노드가 동일한 블록체인 보유
새 블록 → 모든 노드에 전파
```

**51% 공격**: 전체 노드의 과반수를 장악해야 조작 가능 (현실적으로 불가능)

## 7. 트랜잭션 흐름

### 전체 흐름 요약

```
1. Alice가 Bob에게 10코인 전송 요청
        ↓
2. 트랜잭션이 네트워크에 전파
        ↓
3. 채굴자가 트랜잭션을 블록에 담음
        ↓
4. 채굴 성공 (해시 조건 만족)
        ↓
5. 새 블록이 체인에 추가
        ↓
6. 모든 노드가 동기화
        ↓
7. Bob의 잔액 +10 확정
```

이제 각 단계를 자세히 살펴보겠습니다.

---

### 단계 1: 트랜잭션 생성

**Alice가 Bob에게 10코인 보내기로 결정**

```
┌─────────────────────────────────────────────────────────┐
│                    Alice의 지갑                          │
├─────────────────────────────────────────────────────────┤
│  잔액: 50 코인                                          │
│                                                         │
│  [전송 버튼 클릭]                                        │
│  ┌─────────────────────────────────────────────────┐   │
│  │  받는 주소: Bob (0x7a3f2...)                     │   │
│  │  보낼 금액: 10 코인                              │   │
│  │  수수료: 0.001 코인                              │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**트랜잭션 데이터 생성:**

```
Transaction = {
    sender:     "Alice 주소 (0x3f8a...)",
    recipient:  "Bob 주소 (0x7a3f...)",
    amount:     10,
    timestamp:  "2025-02-04T10:30:00",
    signature:  null  ← 아직 서명 안 됨
}
```

**수식으로 표현:**

```
TX = (Sender, Recipient, Amount, Timestamp)
TX = (Alice, Bob, 10, T₀)
```

---

### 단계 2: 트랜잭션 서명

**Alice가 개인키로 서명 → 본인 증명**

```
┌─────────────────────────────────────────────────────────┐
│                      서명 과정                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   트랜잭션 데이터                                        │
│        │                                                │
│        ▼                                                │
│   ┌─────────┐                                           │
│   │ SHA-256 │  → 해시값 (32바이트)                       │
│   └─────────┘                                           │
│        │                                                │
│        ▼                                                │
│   ┌───────────────────┐                                 │
│   │  ECDSA 서명 알고리즘 │ ← Alice의 개인키               │
│   └───────────────────┘                                 │
│        │                                                │
│        ▼                                                │
│   서명값 (r, s)                                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**수식으로 표현:**

```
H = SHA256(TX)                    # 트랜잭션 해시
Sig = ECDSA_Sign(H, PrivKey_Alice) # 개인키로 서명
```

**서명의 의미:**
- "이 트랜잭션은 Alice가 만든 게 맞다"
- "Alice만 가진 개인키로 서명했으니 위조 불가"

**완성된 트랜잭션:**

```
Signed_TX = {
    sender:     "0x3f8a...",
    recipient:  "0x7a3f...",
    amount:     10,
    timestamp:  "2025-02-04T10:30:00",
    signature:  "0xr...s...",        ← 서명 추가됨
    publicKey:  "0x04abc..."         ← 검증용 공개키
}
```

---

### 단계 3: 네트워크 전파 (Broadcasting)

**서명된 트랜잭션이 모든 노드에 퍼짐**

```
                    Alice의 노드
                         │
                         │ 트랜잭션 전송
                         ▼
            ┌────────────┼────────────┐
            │            │            │
            ▼            ▼            ▼
        ┌──────┐    ┌──────┐    ┌──────┐
        │노드 A│    │노드 B│    │노드 C│
        └──────┘    └──────┘    └──────┘
            │            │            │
            └─────┬──────┴─────┬──────┘
                  ▼            ▼
              ┌──────┐    ┌──────┐
              │노드 D│    │노드 E│
              └──────┘    └──────┘

        ※ 수 초 내에 전 세계 노드에 전파
```

**각 노드가 수행하는 검증:**

```
┌─────────────────────────────────────────────────────────┐
│                    검증 체크리스트                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ✓ 서명이 유효한가?                                      │
│    → ECDSA_Verify(H, Sig, PubKey_Alice) == True         │
│                                                         │
│  ✓ Alice 잔액이 충분한가?                                │
│    → Balance(Alice) ≥ Amount + Fee                      │
│    → 50 ≥ 10 + 0.001 ✓                                  │
│                                                         │
│  ✓ 이중 지불이 아닌가?                                   │
│    → 같은 TX가 이미 처리되지 않았는지 확인                │
│                                                         │
│  ✓ 금액이 양수인가?                                      │
│    → Amount > 0 ✓                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**검증 통과 시:**

```
트랜잭션 → "멤풀(Mempool)"에 추가
           (채굴 대기 중인 트랜잭션 저장소)
```

### 단계 4: 채굴자가 블록에 포함

**채굴자가 멤풀에서 트랜잭션을 선택해 블록 구성**

```
┌─────────────────────────────────────────────────────────┐
│                      Mempool (대기열)                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  TX₁: Alice → Bob (10코인)     수수료: 0.001           │ ← 선택됨
│  TX₂: Carol → Dave (5코인)     수수료: 0.002           │ ← 선택됨
│  TX₃: Eve → Frank (20코인)     수수료: 0.0005          │ ← 보류
│  TX₄: Grace → Henry (3코인)    수수료: 0.003           │ ← 선택됨
│  ...                                                    │
│                                                         │
│  ※ 채굴자는 보통 수수료 높은 순으로 선택                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                   새 블록 후보                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Block #1001 (후보)                                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  previous_hash: 0000abc... (Block #1000의 해시)  │   │
│  │  timestamp: 2025-02-04T10:30:15                  │   │
│  │  transactions: [TX₁, TX₂, TX₄]                   │   │
│  │  nonce: ???  ← 아직 모름                         │   │
│  │  hash: ???   ← 채굴해야 알 수 있음               │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 단계 5: 채굴 (작업 증명)

**조건을 만족하는 Nonce 찾기**

```
목표: 해시가 "0000"으로 시작 (난이도 4)

┌─────────────────────────────────────────────────────────┐
│                      채굴 과정                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  시도 1:  Nonce = 0                                     │
│           Hash = SHA256(블록데이터 + 0)                  │
│                = "7f3a2b1c..."  ❌ 0으로 안 시작         │
│                                                         │
│  시도 2:  Nonce = 1                                     │
│           Hash = SHA256(블록데이터 + 1)                  │
│                = "e5c8d1f0..."  ❌                       │
│                                                         │
│  시도 3:  Nonce = 2                                     │
│           Hash = SHA256(블록데이터 + 2)                  │
│                = "2a9f4e7b..."  ❌                       │
│                                                         │
│  ...  (수만~수십만 번 반복)                              │
│                                                         │
│  시도 78,803:  Nonce = 78803                            │
│                Hash = SHA256(블록데이터 + 78803)         │
│                     = "0000c2f8a1b3..."  ✅ 성공!        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**수식으로 표현:**

```
목표: Hash < Target

Target = 0000FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF
         (앞에 0이 4개 = 난이도 4)

채굴:
  for nonce in 0, 1, 2, ...:
      H = SHA256(PrevHash || Timestamp || TXs || nonce)
      if H < Target:
          성공! break
```

**확률 계산:**

```
난이도 4 = 해시가 0000으로 시작해야 함
         = 16진수 4자리가 0
         = 확률 1/16⁴ = 1/65,536

→ 평균 65,536번 시도 필요
```

### 단계 6: 블록 완성 및 전파

**채굴 성공! 완성된 블록:**

```
┌─────────────────────────────────────────────────────────┐
│                   Block #1001 (완성)                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  previous_hash: 0000abc123def456...                     │
│  timestamp:     2025-02-04T10:30:15                     │
│  nonce:         78803                                   │
│  hash:          0000c2f8a1b3e4d5...                     │
│                                                         │
│  transactions:                                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │ TX₁: Alice → Bob      10코인                    │   │
│  │ TX₂: Carol → Dave      5코인                    │   │
│  │ TX₄: Grace → Henry     3코인                    │   │
│  │ TX₀: SYSTEM → Miner  100코인 (채굴보상)          │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**네트워크 전파:**

```
                    채굴자 노드
                         │
                         │ "새 블록 발견!"
                         ▼
```

### 단계 8: 잔액 업데이트

**트랜잭션 반영 후 최종 잔액**

```
┌─────────────────────────────────────────────────────────┐
│                      잔액 변화                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Alice:                                                 │
│    Before: 50 코인                                      │
│    - 전송: 10 코인                                      │
│    - 수수료: 0.001 코인                                 │
│    ─────────────────                                    │
│    After:  39.999 코인                                  │
│                                                         │
│  Bob:                                                   │
│    Before: 20 코인                                      │
│    + 수신: 10 코인                                      │
│    ─────────────────                                    │
│    After:  30 코인  ✓                                   │
│                                                         │
│  Miner:                                                 │
│    Before: 0 코인                                       │
│    + 채굴보상: 100 코인                                 │
│    + 수수료: 0.006 코인 (3건 합계)                      │
│    ─────────────────                                    │
│    After:  100.006 코인                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**수식으로 표현:**

```
Balance(Alice)_after = Balance(Alice)_before - Amount - Fee
                     = 50 - 10 - 0.001
                     = 39.999

Balance(Bob)_after   = Balance(Bob)_before + Amount
                     = 20 + 10
                     = 30

Balance(Miner)_after = Balance(Miner)_before + Reward + Σ(Fees)
                     = 0 + 100 + 0.006
                     = 100.006
```

### 컨펌(Confirmation) 이해하기

**트랜잭션이 확정되는 과정**

```
Block 1001에 TX 포함됨 → 1 컨펌
        ↓
Block 1002 추가됨     → 2 컨펌
        ↓
Block 1003 추가됨     → 3 컨펌
        ↓
Block 1004 추가됨     → 4 컨펌
        ↓
Block 1005 추가됨     → 5 컨펌
        ↓
Block 1006 추가됨     → 6 컨펌 ✓ (일반적으로 확정 간주)
```

**왜 6컨펌인가?**

```
공격자가 TX를 되돌리려면:
  - Block 1001부터 다시 채굴해야 함
  - 6개 블록 = 약 60분치 채굴량 (비트코인 기준)
  - 전체 네트워크의 51% 이상 해시파워 필요
  - 현실적으로 불가능 → "확정"으로 간주
```

## 10. 용어집 (Glossary)

### 기본 용어

| 용어 | 영문 | 설명 |
|------|------|------|
| **제네시스 블록** | Genesis Block | 블록체인의 첫 번째 블록. 이전 블록이 없어 previous_hash = "0". 모든 체인의 시작점 |
| **노드** | Node | 블록체인 네트워크에 참여하는 컴퓨터. 블록체인 사본을 보유하고 검증 수행 |
| **피어** | Peer | 네트워크에서 동등한 위치의 다른 노드. P2P = Peer to Peer (동등한 노드 간 통신) |
| **지갑** | Wallet | 암호화폐를 보관하는 소프트웨어. 실제로는 개인키/공개키 쌍을 관리 |
| **주소** | Address | 암호화폐를 주고받는 계좌번호. 공개키에서 파생된 문자열 |

### 트랜잭션 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **트랜잭션** | Transaction (TX) | 한 주소에서 다른 주소로의 가치 이동 기록 |
| **펜딩** | Pending | 아직 블록에 포함되지 않은 대기 중인 트랜잭션 |
| **컨펌** | Confirmation | 트랜잭션이 블록에 포함된 후 추가된 블록 수. 6컨펌 = 거의 확정 |
| **수수료** | Fee / Gas | 트랜잭션 처리 대가로 채굴자에게 지불하는 금액 |
| **UTXO** | Unspent TX Output | 아직 사용되지 않은 트랜잭션 출력값 (비트코인 잔액 모델) |

### 채굴/합의 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **채굴** | Mining | 새 블록을 생성하고 보상받는 과정 |
| **채굴자** | Miner | 채굴을 수행하는 노드 또는 사람 |
| **난이도** | Difficulty | 채굴 조건의 엄격함. 높을수록 더 많은 연산 필요 |
| **해시레이트** | Hash Rate | 초당 해시 계산 횟수. 채굴 성능 지표 (TH/s = 초당 1조 해시) |
| **블록 보상** | Block Reward | 채굴 성공 시 받는 코인. 비트코인은 4년마다 반감 |
| **반감기** | Halving | 블록 보상이 절반으로 줄어드는 시점 (비트코인: 21만 블록마다) |

### 합의 알고리즘

| 용어 | 영문 | 설명 |
|------|------|------|
| **PoW** | Proof of Work | 작업 증명. 연산력으로 채굴 (비트코인) |
| **PoS** | Proof of Stake | 지분 증명. 보유량에 비례해 채굴 권한 (이더리움 2.0) |
| **DPoS** | Delegated PoS | 위임 지분 증명. 대표자를 뽑아 블록 생성 (EOS) |
| **합의** | Consensus | 분산된 노드들이 같은 상태에 동의하는 과정 |
| **포크** | Fork | 체인이 두 갈래로 분기. 소프트/하드 포크 구분 |

### 보안 관련

| 용어 | 영문 | 설명 |
|------|------|------|
| **51% 공격** | 51% Attack | 과반수 해시파워로 체인 조작 시도. 이론적 취약점 |
| **이중 지불** | Double Spending | 같은 코인을 두 번 사용하려는 시도. 블록체인이 방지 |
| **개인키** | Private Key | 암호화폐 소유 증명. 절대 노출 금지 |
| **공개키** | Public Key | 개인키에서 파생. 주소 생성에 사용 |
| **서명** | Signature | 개인키로 트랜잭션에 서명해 소유권 증명 |

### 고급 개념

| 용어 | 영문 | 설명 |
|------|------|------|
| **스마트 컨트랙트** | Smart Contract | 블록체인에서 실행되는 자동화된 계약 코드 (이더리움) |
| **토큰** | Token | 기존 블록체인 위에서 발행된 디지털 자산 (ERC-20 등) |
| **디앱** | DApp | 분산 애플리케이션. 블록체인 기반 앱 |
| **오라클** | Oracle | 블록체인 외부 데이터를 가져오는 서비스 |
| **레이어2** | Layer 2 | 메인 체인 위에 구축된 확장 솔루션 (라이트닝 네트워크) |
