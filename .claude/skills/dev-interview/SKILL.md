# Dev Interview Skill

인터뷰 방식으로 새 프로젝트를 체계적으로 생성하는 skill이다.

## 버전 히스토리

### v2.0 (2026-01-06)
- **템플릿 동기화**: tpl_project_claude.md와 완전 일치
- **인터뷰 질문 추가**: 완료 조건, 관련 Area/Resource 수집
- **tasks.md 개선**: Phase 기반 구조 + 번호 체계
- **경로 표기 규칙 준수**: `/` → `>>` 변경
- **산출물 YAML 기본값**: "예 (권장)"으로 변경
- **Troubleshooting 섹션** 추가
- **워크플로우 통합 가이드** 추가

### v1.0 (초기)
- 4 Phase 인터뷰 흐름
- 4가지 프로젝트 유형별 템플릿
- AskUserQuestion 도구 기반 대화형 생성

## Purpose

이 skill은 사용자와 대화형 인터뷰를 통해:
- 프로젝트 목표와 범위를 명확히 정의
- 적절한 프로젝트 유형 선택
- 필요한 자원과 제약사항 파악
- 표준화된 프로젝트 구조 자동 생성
- CLAUDE.md에 프로젝트 컨텍스트 기록
- **완료 정의(Done Criteria) 설정**
- **관련 Area/Resource 연결**

## When to Use

다음과 같은 사용자 요청이 있을 때 이 skill을 사용한다:

- "새 프로젝트 만들어줘"
- "프로젝트 시작하자"
- "/dev-interview"
- "XXX 분석 프로젝트 만들어줘"
- "XXX 기획 시작해줘"

## Interview Flow

### Phase 1: 프로젝트 기본 정보

```
질문 1: 프로젝트 이름
├── header: "프로젝트명"
├── question: "프로젝트 이름을 입력해주세요 (예: AI전략수립, 시장분석)"
└── type: 자유 입력
```

```
질문 2: 프로젝트 유형
├── header: "유형"
├── question: "어떤 유형의 프로젝트인가요?"
├── options:
│   ├── "분석/리서치" - 데이터 분석, 시장조사, 벤치마킹
│   ├── "기획/전략" - 업무계획, 전략수립, 정책개발
│   ├── "문서화/보고서" - 보고서 작성, 가이드 제작
│   └── "개발/구현" - 코드 개발, 시스템 구축
└── multiSelect: false
```

```
질문 3: 프로젝트 목표 및 완료 정의
├── header: "목표"
├── question: "이 프로젝트의 핵심 목표와 완료 조건을 알려주세요."
├── hint: "예: 목표 - AI 전략 보고서 완성 / 완료 조건 - 1) 보고서 작성 2) 경영진 보고"
└── type: 자유 입력 (목표 1-2문장 + 완료 조건 1-3개)
```

### Phase 2: 범위 및 산출물

```
질문 4: 주요 산출물
├── header: "산출물"
├── question: "어떤 산출물이 필요한가요?"
├── options:
│   ├── "분석 보고서" - 데이터 기반 인사이트 문서
│   ├── "실행 계획서" - 단계별 액션 플랜
│   ├── "프레젠테이션" - 발표 자료
│   └── "시각화 자료" - 차트, 인포그래픽
└── multiSelect: true
```

```
질문 5: 기존 자료
├── header: "자료"
├── question: "활용할 기존 자료가 있나요?"
├── options:
│   ├── "있음 - 폴더/파일 지정" - 자료 경로 입력
│   ├── "있음 - 나중에 추가" - 빈 sources 폴더 생성
│   └── "없음 - 새로 수집" - 자료 수집부터 시작
└── multiSelect: false
```

```
질문 6: 관련 Area/Resource (NEW in v2.0)
├── header: "연결"
├── question: "이 프로젝트와 연결할 Area나 Resource가 있나요?"
├── hint: "예: A-work, R-about_ibk"
├── options:
│   ├── "있음 - 직접 입력" → Area/Resource 경로 입력
│   ├── "자동 추천받기" → 프로젝트 목표 기반 자동 제안
│   └── "없음/나중에" → 빈 값으로 진행
└── multiSelect: false
```

### Phase 3: 산출물 설정

```
질문 7: 산출물 YAML 메타데이터
├── header: "YAML"
├── question: "산출물에 YAML 메타데이터를 추가할까요?"
├── options:
│   ├── "예 (권장)" - Dataview 검색, 버전 추적, 프로젝트 연결 가능
│   └── "아니오" - 간단하게 유지 (단순 메모 수준 산출물)
├── default: "예 (권장)"
└── multiSelect: false
```

> **참고**: YAML 메타데이터를 추가하면 볼트 전체에서 산출물을 검색하고 관리할 수 있습니다.
> 템플릿: `9_Templates >> tpl_project_output.md`

### Phase 4: 일정 및 우선순위

```
질문 8: 마감일
├── header: "마감"
├── question: "마감일이 있나요?"
├── options:
│   ├── "있음" - 날짜 입력 요청
│   └── "없음/유동적" - 마감 없이 진행
└── multiSelect: false
```

```
질문 9: 우선순위
├── header: "우선순위"
├── question: "이 프로젝트의 우선순위는?"
├── options:
│   ├── "높음" - 즉시 착수, 집중 작업
│   ├── "중간" - 일반적인 진행
│   └── "낮음" - 여유있게 진행
└── multiSelect: false
```

## Project Structure Generation

인터뷰 완료 후 다음 구조를 자동 생성한다:

```
1_Projects >> P-{YYYYMMDD}_{프로젝트명}/
├── CLAUDE.md          # 프로젝트 컨텍스트 (인터뷰 내용 포함)
├── sources/           # 원본 자료
├── notes/             # 정리된 노트
├── outputs/           # 산출물
├── docs/              # 작업 문서
└── tasks.md           # 태스크 목록
```

## Best Practices

1. **질문 최소화**: 필수 정보만 수집, 상세 사항은 나중에 보완
2. **유연한 진행**: 사용자가 "빠르게"/"간단히" 요청 시 Phase 2-4 생략 가능
3. **컨텍스트 보존**: 모든 인터뷰 내용을 CLAUDE.md에 기록
4. **즉시 시작**: 프로젝트 생성 후 첫 번째 태스크 바로 시작 제안
5. **완료 정의 명확화**: 모호한 완료 조건은 구체화 요청

## 워크플로우 통합

이 스킬로 생성된 프로젝트는 다음 스킬들과 함께 사용할 때 효과적이다:

### 문서 변환 파이프라인
```
1. markitdown CLI로 PDF/DOCX → Markdown 변환
2. md-cleaner 스킬로 클리닝 (sources/ 또는 notes/ 저장)
```

### 시각화 파이프라인
```
1. 분석 완료 후 데이터 테이블 작성
2. make-chart 스킬로 차트 생성 (9_Attachments >> charts >> AI_generates/)
3. make-infographic 스킬로 인포그래픽 생성 (복합 시각화)
```

### IBK 경영지표 분석 프로젝트
```
1. dev-interview로 프로젝트 생성 (유형: 분석/리서치)
2. R-about_ibk 연결 설정
3. textbook-analyzer 에이전트로 심층분석 보고서 생성
```

### 외부 데이터 수집 프로젝트
```
1. dev-interview로 프로젝트 생성
2. api-* 스킬로 데이터 수집 (api-apt, api-fisis, api-ecos 등)
3. make-chart로 시각화
```

## 문제 해결

### 동일 이름 프로젝트 충돌

**증상**: 같은 날짜에 동일한 이름의 프로젝트가 이미 존재

**해결**:
1. 기존 프로젝트 확인: `ls 1_Projects >> P-YYYYMMDD_{이름}`
2. 이름 변경 제안: `{이름}_v2` 또는 세부 설명 추가
3. 기존 프로젝트 아카이브 후 재생성

### 인터뷰 중단 시

**증상**: 인터뷰 도중 세션이 끊기거나 중단됨

**해결**:
1. 이미 생성된 폴더 확인
2. 부분 생성된 경우: CLAUDE.md와 tasks.md 수동 완성
3. 미생성된 경우: 인터뷰 재시작

### 관련 Area/Resource 찾기 어려움

**증상**: 사용자가 연결할 Area/Resource를 모름

**해결**:
1. 프로젝트 목표 키워드 기반 자동 추천
2. `2_Areas/` 및 `3_Resources/` 폴더 목록 제시
3. "나중에 추가" 선택 후 진행

### 완료 조건 작성 어려움

**증상**: 사용자가 완료 조건을 구체화하지 못함

**해결**:
1. 프로젝트 유형별 기본 완료 조건 템플릿 제안:
   - **분석/리서치**: "분석 보고서 완성", "인사이트 도출", "발표/공유"
   - **기획/전략**: "전략서 완성", "경영진 보고", "피드백 반영"
   - **문서화/보고서**: "문서 완성", "검토 완료", "배포"
   - **개발/구현**: "기능 구현", "테스트 통과", "배포"
2. 사용자에게 선택 또는 수정 요청
